The T-Panel

The T-Panel is Better Way Lighting's upcoming new panel light whose design and software was created in-house. The T-Panel is Better Way Lighting's first panel light that is made to be comparable to other panel lights in white output but is lighter, easier to rig, and less expensive. The T-Panel is also designed to fit accessories made for other panel lights of the same size.

Since the T-Panel has not been released at ny the time I finished my co-op, several details about the panel have been omitted, including the official name of the panel.

I worked on the design, layout, and behaviour of the T-Panel's UI, as well as connecting the UI to the data model and device backend. I also added new data model entries and backend functionality as needed to implement new features or modify existing features and UI behaviour.

UI Design and Features

Initial UI Design and Layout

I started designing the UI by first making a list of all the potential features and UI screens we wanted to include based on the reference panel lights and the T-Panel specifications. Using the list, I designed mockups of the UI cards and layout in Figma based on the UI library used on our other devices that use the smaller variant of the T-Panel screen.

Feature and Layout Selection

After going through the UI mockups with the team, we decided on the device and UI features we wanted to include from the UI mockups, as well as the design of the UI cards. Based on the new feature list, I created flow charts for each of the potential UI layouts. Using the layout flow charts, the team decided on a single menu layout that updates to show control mode-specific settings based on the selected control mode.

Using the UI mockups, I test-coded test versions of the UI cards with and without the menu title to roughly determine the layout of the cards and the font sizes. Based on the feedback from the team, we determined the layout of the card elements and chose font sizes that allowed us to fit at least 3 lines of text on the display. Based on the test versions of the UI cards, I created a mockup list of the basic card types and font size options. The card type mockup list was then sent to the design team to finalize the design of the UI cards.

Building the UI

Initial UI Coding

Using the selected UI layout and the finalized UI card designs, I began modifying the UI library on a separate branch to fit the larger screen size and coding the T-Panel UI in a separate code repository while the backend and data model was being built. I also imported and modified existing custom cards we needed from the other devices using the same UI library and created some additional new custom cards. While coding the T-Panel UI, I also added dummy callback functions to the cards that require them in preparation for connecting the UI with the data model and backend.

Linking the UI

Once the initial UI, data model, and backend were created, I began linking the UI to the data model and backend. During the linking process, the UI was reorganized and the UI cards for features not included in the initial release were removed. Additionally, I added new UI cards and display elements for features in the initial backend that wasn't in the initial UI. I also assisted in adding any data model entries and backend functionality for the UI cards that had not been implemented yet.

Adding New Features

After linking the initial UI to the initial data model and backend, the team began implementing new major features and adjusting the behaviour of existing features and the UI. Once the major features were added, we then began adding the minor features we wanted to have for release. I assisted in modifying and adding to the UI, data model, and backend based on the features and device behaviours we wanted. As of the writing of this report, the final remaining features for the T-Panel release are being added.

Some of the features I added or assisted in adding the UI, data model entries and backend for include the adjustable lock timeout, several warning and status modals, navigating to the status screen on encoder scroll and UI lock, device temperature display, and allowing users to set the flash button output to the current light output. I also helped add development-only behaviour and UI cards to make it easier to test the device.
