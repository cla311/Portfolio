Project Details

Date: October - December 2018

Project Type: School Project, Team of 2 - Web Design, Full Stack Development

Role: Web Designer, Full Stack Developer

Tools: HTML5, CSS3, PHP, MySQLi, Ajax, XAMPP, phpMyAdmin

Goal/Description

In a team of two, create a PHP website that uses external data stored in a MySQLi database and provides additional features for registered users. We chose to create a NHL stats website using the NHL API.

Process

The project began with the project proposal, followed by the Entity Relationship Diagram, which outlined the structure of the website and the database tables needed. Next we began designing the database and main pages of the website. I created the PHP API calls to the NHL API to retrieve the team and player stats, as well as the MySQLi queries to insert stats of new players into the database and to update the stats of existing players. I was also in charge of designing the Ajax filters and pagination for the search page, as well as creating the database tables and MySQLi queries to retrieve the stats.

Challenges

One challenge I ran into was accessing the data from the NHL API and updating the database efficiently. This occurred because there was no official documentation from the NHL and the only documentation of the NHL API was from an ongoing documentation project created by Drew Hynes with community input on GitLab. I was able to decrease the database update time by updating only the player list in the search page, and then updating the team and player stats and details when the user views details of the specific player/team. This however made the search filters inaccurate as the stats would not be updated.

Retrieving the data was difficult as data was spread out between separate API endpoints. I solved the data retrieval issue by finding out that loading the NHL teams list give the individual team IDs which could then be used to retrieve the team details, stats, and rosters. From the rosters, I was then able to retrieve the individual player IDs to get the player details and stats.

Results

The resulting website was noticeably faster and did not time out when updating the database. The pagination and filters on the search page resulted in smoother, cleaner search experience, allowing users to make more detailed searches, and large search results lists to be viewed more easily in paged results instead of one single list.

Reflection

Through this project, I learned how to use PHP, MySQLi, and Ajax. I also applied my web design and web programming skills, resulting in a simple to use NHL stats database web application. However, the database updating could have been better optimized to make the user experience smoother. In the future, I could solve this issue by using cron/Task Scheduler to update the database periodically. This would result in data not always being up-to-date, but would reduce loading times drastically.
