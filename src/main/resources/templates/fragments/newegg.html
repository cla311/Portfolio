Project Details

Date: February - March 2019

Tools: HTML5, CSS3, JavaScript

Project Type: School Project - Front End Web Design

Team Members: Alex S.

Role: Programmer, UI Designer

Goal/Description

In a team of two, create a prototype responsive e-commerce company website using HTML and CSS. The company we chose was Newegg Canada, a tech-focused e-retailer.

Process

The project began with the consolidation and finalization of the website's style, which could be based on the styles we created for the previous individual project. We settled on using the style I created with a couple minor changes. Next we began designing the main pages of the website. Using flexboxes, I created a responsive design that allowed the website to fit various device screen sizes. I also designed the checkout form, the product page image gallery, and the product details page.

Challenges

One challenge I ran into was designing the flexboxes so they would fit the way we wanted on different screen sizes, especially for the product page image gallery. Initially, the image gallery would scale responsively between certain screen sizes, but would then snap to abruptly to the next style when the screen size became smaller or bigger than the initial range. I eventually solved this problem by adjusting the flexbox style starting from the largest screen size the website would support, which made sure the smaller styles would not break the larger ones.

Results

The resulting website had a clean and simple visual style that fit well on various screen sizes. The flow of the website resulted in a smooth navigation experience, as each step in the purchasing process (viewing products, adding to cart, and checking out) is spread out between 1-2 pages and requires minimal clicks to complete (not including entering checkout details).

Reflection

Through this project, I gained a better understanding of how to create a smooth website flow, and how to design flexboxes to better fit different screen sizes. This project also helped me further develop my skills in using JavaScript and using flexboxes. One thing that could be changed in the future is the layout of the website. Currently the main body content fills the entire window width, which makes it too wide on larger screens. In the future, I would change the width main body content's flexboxes and make them smaller, and adjust the breakpoints and wrapping accordingly.
